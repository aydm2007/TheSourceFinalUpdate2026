# AstMutexLockManager

## Purpose
The **AstMutexLockManager** provides a lightweight mutex mechanism for AST‑level file edits. It ensures that multiple agents or parallel swarm tasks can safely acquire exclusive write access to a specific range of lines in a source file, preventing Git‑style conflicts and race conditions.

## API
```js
AstMutexLockManager.handler({
  agent_id: string,          // Unique identifier for the requesting agent
  file_path: string,         // Absolute path to the target source file
  line_range: number[]       // Inclusive line numbers to lock, e.g. [10, 25]
});
```
- **Returns** a promise that resolves when the lock is acquired.
- The lock is automatically released when the promise resolves or rejects.

## Usage Example
```js
await AstMutexLockManager.handler({
  agent_id: 'agent-42',
  file_path: 'src/utils/helpers.js',
  line_range: [120, 135]
});
// Perform safe AST edit here
```

## When to Use
- Parallel swarm operations that modify the same file.
- Automated refactoring tools that generate AST patches.
- Any scenario where concurrent file writes could cause merge conflicts.

## Best Practices
1. Keep the locked range as narrow as possible.
2. Release the lock promptly after the edit.
3. Combine with `SurgicalDiff` or `ASTAutoPatch` for atomic changes.
4. Log lock acquisition/release for auditability (automatically recorded in `shadow_ledger.jsonl`).
